## Description
Implement per-shop SePay configuration. Each shop must have its own SePay config for production invoice issuing.

## Requirements
- Store env, provider account, template code, invoice series, default payment method and default tax rate per shop.
- Store client ID/secret encrypted.
- Per-shop SePay config required for production. No tenant-level fallback when issuing production invoices.
- Sandbox/dev: tenant-level config can be used as template for faster onboarding.
- Return masked config only (has_client_secret flag, last_updated_at, no raw secrets).

## Config Resolution

```
Production issue:

  -> If missing: BLOCK with error SHOP_SEPAY_CONFIG_REQUIRED

Sandbox/dev create draft:
  -> Prefer shop-level config if exists
  -> Fall back to tenant-level config if shop-level missing (for onboarding convenience)
  -> This fallback is only for sandbox/dev, NOT for production issue
```
