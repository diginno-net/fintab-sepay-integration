## Description
Build login and dashboard screens. Login authenticates user and returns tenant + shops list. Dashboard shows overview of configured shops and invoice activity.

## Requirements
- Login form: email + password, loading state on submit, inline error on failure
- After successful login, call GET /v1/me to get tenant + shops list
- Redirect to /onboarding if no shops configured yet
- Redirect to /dashboard if shops exist
- Dashboard uses asymmetric bento layout
- Dashboard shows: shops with config status, running jobs, drafts, issued invoices, recent failures
- Empty dashboard state: "Chưa có cửa hàng nào" -> CTA to /shops/new or onboarding
- Shop switcher component in dashboard header for multi-shop users

## Login Flow

```
1. User submits email/password
2. POST /v1/auth/login -> 200 + session cookie set
3. GET /v1/me -> { user, tenant, shops[], currentShopId }
4. If shops.length === 0: redirect to /onboarding
5. If shops.length > 0: set currentShopId (first or stored preference), redirect to /dashboard
```

## Dashboard Empty State

```
If tenant has no shops:
  -> Show empty state: "Chưa có cửa hàng nào"
  -> CTA: "Thêm cửa hàng" -> /shops/new

If shop exists but no SePay config:
  -> Show shop card with warning: "Chưa cấu hình SePay"
  -> CTA: "Cấu hình ngay" -> /shops/:shopId/settings/sepay
```
