## Description
Implement tax resolution for each Pancake order item before generating SePay invoice preview payload.

## Requirements
- Match order item to product catalog by barcode, product code, product id or normalized name.
- Resolve shop-level product tax profile first.
- Fall back to tenant-level product tax profile.
- Fall back to shop tax defaults when policy allows.
- Return tax resolution source per item: `product_profile`, `tenant_profile`, `shop_default`, `manual_override`.
- Return warnings when using defaults or when product/tax profile is missing.
- Block create draft when `unknown_product_policy=block` and an item lacks tax profile.
- Validate GTGT tax_rate is one of `-2`, `-1`, `0`, `5`, `8`, `10`.

## Resolution Order

```text
1. product_tax_profiles by tenant_shop_id + source_product_code
2. product_tax_profiles by tenant_id + source_product_code
3. shop_tax_defaults by tenant_shop_id
4. warning or blocking validation error based on unknown_product_policy
```
