# Task: BE-002 - Harden invoice job handlers for draft, issue and download

## Metadata
- Status: completed

## Description
Củng cố handler nghiệp vụ hóa đơn để status update rõ ràng, response provider được lưu đủ thông tin và download PDF/XML trả lỗi dễ hiểu khi thiếu reference code hoặc provider trả format không mong muốn.

## Requirements
- `handleCreateDraft` giữ hành vi gọi SePay create invoice với `is_draft` enabled.
- Sau khi create draft request thành công, invoice job phải ở trạng thái phù hợp để user check/poll create status.
- `handlePollCreate` phải lưu reference code nếu provider trả về.
- `handleIssue` chỉ chạy khi đã có reference code.
- `handlePollIssue` phải cập nhật invoice number và download availability khi provider báo success.
- `downloadInvoiceArtifact` phải fail rõ ràng nếu thiếu reference code.
- Response/error provider cần được giữ trong `response_json` hoặc `error_json` theo behavior hiện tại.

## Context
Các endpoint liên quan đã tồn tại trong `invoice.routes.ts`. Task này không tạo route mới, chỉ làm handler an toàn và rõ trạng thái hơn để frontend action có thể dựa vào status.
