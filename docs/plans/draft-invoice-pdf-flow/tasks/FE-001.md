## Description
Cập nhật `/jobs` để hỗ trợ hai mode rõ ràng: invoice job detail theo `invoiceJobId` và background job detail theo `jobId`. Invoice job là mode chính cho user thao tác hóa đơn.

## Requirements
- Nếu URL có `invoiceJobId`, page phải gọi API lấy invoice job từ `/v1/invoices/jobs/:id` thông qua helper hiện có.
- Nếu URL có `jobId`, page phải gọi API lấy background job từ `/v1/jobs/:id`.
- Nếu không có id, page vẫn hiển thị recent background jobs như hiện tại.
- Invoice job detail phải hiển thị tối thiểu: order id, status, invoice type, SePay create tracking code, SePay issue tracking code, reference code, invoice number, download availability, mapping warnings, request payload, response JSON.
- Background job detail phải hiển thị tối thiểu: id, type, status, attempts, poll attempts, payload, last error masked nếu có.
- Action area của invoice job phải truyền đúng `invoice_jobs.id` vào `JobActions`.

## Interface Definitions
Invoice job detail data cần xử lý các field dạng object linh hoạt:
- id: string
- source_order_id: string
- status: string
- invoice_type: string
- sepay_create_tracking_code?: string
- sepay_issue_tracking_code?: string
- sepay_reference_code?: string
- invoice_number?: string
- download_available?: boolean
- mapping_warnings_json?: array/object
- request_payload_json?: object
- response_json?: object
