## Description
Cập nhật action buttons cho invoice job: check status, retry, issue, view PDF và download PDF. Thêm helper xử lý response PDF từ backend/SePay linh hoạt.

## Requirements
- `JobActions` nhận `invoiceJobId` là id từ bảng `invoice_jobs`.
- `Check status` gọi endpoint invoice job check status.
- `Retry` gọi endpoint invoice job retry khi status là failed hoặc timeout.
- `Issue` chỉ hiện khi status là `draft_created`.
- `View PDF` và `Download PDF` hiện khi status là `draft_created` hoặc `issued`, nhưng phải xử lý lỗi nếu provider chưa cho tải draft PDF.
- Auto-refresh status định kỳ khi status chưa terminal.
- Helper download phải xử lý tối thiểu các dạng provider response: URL, download URL, file URL, base64, raw string dạng URL/base64, hoặc object nested phổ biến.
- Khi không nhận diện được response PDF, hiển thị message lỗi rõ ràng cho user.

## Interface Definitions
Download helper nên expose hành vi tự nhiên:
- nhận artifact response unknown
- nhận mode: view hoặc download
- nhận filename gợi ý
- mở tab mới khi view
- trigger download khi download
- trả về message lỗi hoặc throw error khi không xử lý được
